# Промышленный микросервис оптимизации и дистрибуции медиа-контента отелей

Проект разработан в рамках производственной практики студентами группы ИВТ-164 : Логуновым Александром и Баскаковым Артёмом

## 🚀 Новый функционал и архитектурные изменения
* **Защита от Path Traversal (Безопасность)**: Внедрена строгая санитизация путей через `os.path.basename()`. Сервер изолирован от уязвимостей чтения произвольных файлов через подмену путей (например, `src=../../etc/passwd`).
* **Полноценный REST API (Загрузка файлов)**: Реализован асинхронный эндпоинт `POST /upload` для программной или ручной загрузки оригиналов изображений на сервер.
* **Мультиформатная конвертация**: Поддерживается динамическое транскодирование в обе стороны между форматами `WebP`, `JPEG` и `PNG` на основе параметра `fmt`.
* **Динамическое квантование (Quality)**: Интегрирован прием параметра `quality` (1–100) из строки запроса для гибкого управления весом файла и качеством картинки.
* **Файловый кэш и Инвалидация**: Автоматический кэш (HIT/MISS) с проверкой TTL (24 часа) и даты модификации оригинала, а также принудительный сброс через метод `DELETE`.

## 🛠️ Стек технологий
* **Язык**: Python 3.11
* **Фреймворк**: FastAPI (ASGI-сервер Uvicorn)
* **Графика**: Pillow (C-оптимизированные алгоритмы, ресайз Lanczos)
* **Инфраструктура**: Docker, Docker Compose (Volumes для маппинга данных)

## 💻 Локальный запуск
1. Установите зависимости: `pip install -r requirements.txt`
2. Запустите асинхронный сервер: `python -m uvicorn main:app --reload`

## 🐳 Развертывание в Docker Compose
Запуск всей изолированной инфраструктуры одной командой:
```bash
docker compose up -d --build
```

## 📊 Документация и Примеры API-запросов

Управлять микросервисом и тестировать валидацию можно через встроенную интерактивную документацию Swagger UI по адресу: `http://localhost:8000/docs`

### 1. Загрузка оригинала (POST)
* **URL**: `http://localhost:8000/upload`
* **Тип данных**: `Multipart/form-data` (поле `file`)

### 2. Динамическая обработка и получение превью (GET)
* **Пример запроса**: `http://localhost:8000/image?w=300&h=200&fmt=webp&quality=80&src=test.jpg`
* **Параметры**:
  * `w`, `h` — целевой размер в пикселях.
  * `fmt` — целевой формат (`webp`, `jpeg`, `png`).
  * `quality` — качество сжатия от 1 до 100 (по умолчанию 80).
  * `src` — имя файла-оригинала.

### 3. Принудительный сброс кэша (DELETE)
* **Пример запроса**: `http://localhost:8000/image?src=test.jpg` (удаляет все нарезанные превью для данного файла).
